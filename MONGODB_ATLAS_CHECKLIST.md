# MongoDB Atlas Setup Checklist

Quick checklist to set up MongoDB Atlas for your Node.js application.

## ‚úÖ Setup Steps

### Step 1: Create Account
- [ ] Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas)
- [ ] Click "Try Free" or "Sign Up"
- [ ] Sign up with Google, GitHub, or Email
- [ ] Verify email (if using email signup)

### Step 2: Create Cluster
- [ ] Choose "M0 FREE" tier
- [ ] Select cloud provider (AWS, Azure, or GCP)
- [ ] Select region (closest to you)
- [ ] Click "Create cluster"
- [ ] Wait 3-5 minutes for cluster creation

### Step 3: Create Database User
- [ ] Go to "Database Access"
- [ ] Click "Add New Database User"
- [ ] Choose authentication method: "Password"
- [ ] Enter username (e.g., `admin`, `myuser`)
- [ ] Set password (use autogenerate or create strong password)
- [ ] **SAVE USERNAME AND PASSWORD** (you won't see password again!)
- [ ] Select user privileges: "Atlas admin"
- [ ] Click "Add User"

### Step 4: Configure Network Access
- [ ] Go to "Network Access"
- [ ] Click "Add IP Address"
- [ ] Choose one:
  - [ ] "Allow Access from Anywhere" (0.0.0.0/0) - Development only
  - [ ] "Add Current IP Address" - Recommended for production
  - [ ] Add specific IP address manually
- [ ] Click "Confirm"

### Step 5: Get Connection String
- [ ] Go to "Database" or "Clusters"
- [ ] Click "Connect" on your cluster
- [ ] Select "Connect your application"
- [ ] Choose Driver: "Node.js"
- [ ] Choose Version: "4.1 or later"
- [ ] Copy connection string
- [ ] Replace `<username>` with your database username
- [ ] Replace `<password>` with your database password
- [ ] Add database name before `?` (e.g., `crud-db`)

### Step 6: Configure Application
- [ ] Create `.env` file in backend directory
- [ ] Add MongoDB connection string:
  ```env
  MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/database-name?retryWrites=true&w=majority
  PORT=5000
  NODE_ENV=development
  ```
- [ ] Save `.env` file
- [ ] Verify `.env` is in `.gitignore`

### Step 7: Test Connection
- [ ] Install dependencies: `npm install`
- [ ] Start server: `npm run dev`
- [ ] Check console for: "MongoDB Connected"
- [ ] Test API endpoints
- [ ] Verify data in MongoDB Atlas

## ‚ö†Ô∏è Important Notes

- **Save credentials:** Username and password are required for connection
- **URL encoding:** Special characters in password must be URL-encoded
  - `@` ‚Üí `%40`
  - `#` ‚Üí `%23`
  - `$` ‚Üí `%24`
  - `%` ‚Üí `%25`
- **IP whitelist:** Add your IP address or use `0.0.0.0/0` for development
- **Security:** Never commit `.env` file to version control
- **Database name:** Will be created automatically when you insert data

## üêõ Troubleshooting

### Connection Timeout
- [ ] Check IP address is whitelisted
- [ ] Verify internet connection
- [ ] Check if cluster is running

### Authentication Failed
- [ ] Verify username and password
- [ ] URL-encode special characters in password
- [ ] Check database user exists

### Network Access Denied
- [ ] Add IP address to whitelist
- [ ] Use `0.0.0.0/0` for development
